вҸә Excellent catch вҖ” yes, you've spotted the same prior in both. Steve's soft
  binning (and Faraz's e^-d accumulate weights вҖ” he does it too) and CS temporal
  regularization both encode one belief: neighboring respiratory phases look 
  similar. The difference is where and how that belief is applied вҖ” and it's the
  difference that matters.

  Steve/Faraz: sharing in the data domain (open-loop)

  binwt = exp(-bindistВІ/2) mixes neighboring phases' k-space before recon. This
  is what the dynamic-MRI literature calls view sharing (sliding window / KWIC).
  Properties:

  - Fixed Gaussian width, hand-tuned (bindist0sq=2), applied unconditionally вҖ”
  every voxel, every region, blurred in time by the same amount
  - The temporal smoothing is baked into the measurement вҖ” after mixing, bin 5's
  k-space genuinely contains bin 4 and 6's anatomy. No downstream step can ever
  ask "was that diaphragm position real or borrowed?"
  - Buys sampling density and SNR per bin; pays with temporal blur you can't
  localize or undo

  CS: sharing in the image domain (closed-loop)

  min_ПҒ  ОЈ_b вҖ–A ПҒ_b вҲ’ s_bвҖ–ВІ  +  О»В·вҖ–D_t ПҒвҖ–

  Each bin's data term uses only that bin's own samples вҖ” unmixed. The
  similarity belief enters as a penalty the solver must trade against data fit:

  - Adaptive: where the data clearly says "the diaphragm moved" (strong signal
  disagreement), data consistency overrules the prior вҶ’ edge stays temporally
  sharp. Where data is ambiguous (noisy uniform region), prior wins вҶ’ noise
  suppressed. Steve's scheme can't make this distinction вҖ” it blurs the
  diaphragm exactly as much as the noise.
  - Tunable and auditable: one О» knob, and you can recon with О»=0 to see exactly
  what the prior added. View-shared data has no О»=0.
  - Choice of D_t is physics: temporal TV (preserves sharp inhale-exhale
  transitions), low-rank (breathing is a few global modes вҖ” very apt for
  ventilation), or temporal-Fourier sparsity (periodic breathing).

  The one-line distinction

  Steve answers "bin 5 is undersampled" by changing the question (recon a
  temporally smoothed object that no longer is bin 5). CS keeps the question and
  constrains the answer. Same prior, open-loop in k-space vs closed-loop in
  image space with the data holding veto power.

  regularization вҖ” soft binning is the poor man's version of it, the best

вқҜ what is D3 and what is CG?                                                    

вҸә D3 вҖ” my label, nothing standard. The handoff doc (handoff-report.md) numbers
  this session's decisions D1вҖ“D6. D3 = "static first; bins are weights, not
  architecture" вҖ” i.e. the API is recon(traj, data, sample_weights=None), static
  = no weights, binned = 16 calls with weight vectors. When the code comments
  say "per handoff D3" they point at that decision record.

  CG = Conjugate Gradient вҖ” the iterative solver that turns the adjoint into an
  actual inverse. The missing piece between gridding and CS:

  Recall from Physics_Notes В§4: measurement is s = AПҒ. Gridding gives you Aбҙҙs
  (the adjoint) вҖ” blurry, density-biased, because AбҙҙA вү  identity for non-uniform
  sampling. The honest recon solves least squares:

  min_ПҒ вҖ–AПҒ вҲ’ sвҖ–ВІ    вҹә    AбҙҙA ПҒ = Aбҙҙs      ("normal equations")

  That's a huge linear system: ПҒ has 10вҒ¶ unknowns, so AбҙҙA is a 10вҒ¶Г—10вҒ¶ matrix вҖ”
  can't form it, can't invert it directly. But you can multiply by it: AбҙҙAВ·x =
  one forward NUFFT + one adjoint NUFFT (seconds). CG is the classic algorithm
  for exactly this situation: solves Mx = b using only multiply-by-M, never M
  itself. Each iteration picks a clever search direction (conjugate to all
  previous ones вҖ” hence the name), guaranteeing fastest possible progress for a
  symmetric positive system. Typically 10вҖ“30 iterations вүҲ converged.

  What it looks like in asap_recon.py:cg_recon: the loop with normal_op(p) (the
  A then Aбҙҙ application), alpha, residual update вҖ” that's the whole textbook
  algorithm, ~15 lines.

  Why it matters in our ladder:
